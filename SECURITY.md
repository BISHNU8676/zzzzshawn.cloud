# Security Policy

## Supported Versions

We release security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

**Do not** open a public GitHub issue for security vulnerabilities.

Instead, please email **shawndsilva.mail@gmail.com** with:

- A description of the vulnerability
- Steps to reproduce (if applicable)
- Potential impact
- Any suggested fix (optional)

We will acknowledge your report as soon as possible and will keep you updated on the status of any fix. We appreciate your effort to disclose your findings in a responsible manner.

## Security Update Process

- We will assess and triage the report promptly.
- We will work on a fix and coordinate with you before public disclosure where appropriate.
- Security updates will be released as patch versions and documented in release notes or a security advisory.

Thank you for helping keep this project and its users safe.
