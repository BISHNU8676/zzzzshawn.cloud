# Contributing

Thank you for your interest in contributing to this project.

## Code of Conduct

This project adheres to the [Contributor Covenant Code of Conduct](CODE_OF_CONDUCT.md). By participating, you are expected to uphold this code.

## How to Contribute

### Reporting Bugs or Suggesting Features

- Open a GitHub Issue in this repository.
- Use a clear, descriptive title and include steps to reproduce (for bugs) or use cases (for features).

### Submitting Changes

1. **Fork the repository** and clone your fork locally.
2. **Set up the project** — see [DEVELOPMENT.md](DEVELOPMENT.md) for prerequisites and setup.
3. **Create a branch** from `main` (e.g. `git checkout -b fix/typo-readme`).
4. **Make your changes** and ensure `pnpm lint` passes.
5. **Commit** with a clear message (e.g. `fix: correct README link`).
6. **Push** to your fork and open a **Pull Request** against `main`.
7. Describe what you changed and why; link any related issues.

### Development Commands

| Command       | Description                    |
| ------------- | ------------------------------ |
| `pnpm dev`    | Start the development server   |
| `pnpm build`  | Build for production          |
| `pnpm start`  | Start the production server   |
| `pnpm lint`   | Run ESLint                    |

## What We’re Looking For

- Bug fixes and clear, small improvements.
- Documentation fixes (README, comments, types).
- Keeping dependencies and patterns consistent with the existing codebase.

If you’re unsure whether a change fits, open an issue first and we can discuss it.

## Security

Please do **not** report security vulnerabilities in public issues. See [SECURITY.md](SECURITY.md) for how to report them privately.

Thanks for contributing!
